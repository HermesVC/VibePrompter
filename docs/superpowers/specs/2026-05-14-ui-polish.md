# UI/UX Polish & Native Windows Experience — Spec

**Date:** 2026-05-14
**Loop:** Ralph (iterative polish — re-fired until quality bar met)

## Goal

Transform the existing Tauri + React desktop app into a polished, production-grade Windows
desktop experience that feels native, lightweight, modern, and premium. The app already has
backend architecture, AI integration, shortcuts, tray, overlays, settings, modes, and a
command palette. **Goal is NOT to rebuild functionality** — it is to refine UX, UI
consistency, animations, layout, native desktop behavior, interaction quality,
accessibility, visual hierarchy, and performance perception.

## Target feel

* Raycast for Windows
* Linear-quality UX
* Native Windows 11 utility
* AI-first productivity software

The application MUST NOT feel like: a wrapped website, an admin dashboard, a bulky
Electron app, a generic Tailwind template.

## Primary UX adjectives

instant • calm • intelligent • keyboard-first • minimal • premium • focused •
invisible-until-needed

## Windows 11 design language

mica-inspired surfaces • acrylic blur • subtle transparency • rounded corners • soft
elevation • minimal borders • smooth transitions • adaptive dark mode. Avoid: excessive
shadows, loud gradients, bright colors, oversized controls, gaming aesthetics.

## Visual style

Restrained palette: charcoal / graphite / muted slate / soft neutral grays. Accents: soft
blue, muted cyan, subtle purple. Color used only to indicate focus, actions, states.

Typography: Inter or Geist; compact, elegant, highly readable. Hierarchy: large clean
titles, muted secondary labels, compact metadata, subtle helper text.

## Component quality

* **Buttons** — tactile, subtle hover, polished active. Primary: slightly elevated, stronger contrast. Secondary: ghost/minimal. Danger: muted red.
* **Inputs** — resemble command-palette inputs, soft borders, elegant focus glow, compact, keyboard nav, smooth focus transitions, instant validation.
* **Cards & panels** — soft transparency, layered depth, subtle blur, minimal borders. Avoid heavy boxed layouts.

## Command palette (most important)

instantaneous, buttery smooth, premium, distraction-free. Smooth open/close (fade + slight
scale), instant keyboard focus, zero layout shift, fuzzy search, keyboard-only navigation,
compact action rows (icon, title, optional subtitle, keyboard hint), fluid hover states.

## Overlay windows

Appear near cursor intelligently, animate smoothly, avoid edge clipping, multi-monitor
support. Animations: soft fade, slight upward movement, fast response. Never jarring.

## System tray

Native context-menu styling, instant responsiveness, clean status indicators, subtle app
states, Windows 11 density.

## Settings window

Feel like Linear / VS Code / modern native tools — NOT enterprise admin dashboards. Compact
sidebar, smooth section transitions, persistent navigation, instant save feedback,
searchable settings.

## Prompt mode manager

Users feel powerful, in control, creative. Editor like modern IDE / AI playground /
structured prompt engineering. Syntax highlighting, templates, live preview, token
estimation, provider selection, test execution.

## History

Lightweight, useful, non-intrusive. Compact cards, expandable previews, subtle metadata,
clean timestamps. Quick reuse, copy result, restore transformation.

## Animation

Duration 120–220ms. Preferred motions: opacity fade, subtle scale, small translations.
Avoid bounce/elastic/exaggerated. Respect `prefers-reduced-motion`.

## Microinteractions

Hover glow, button press depth, smooth toggles, animated loading, progressive AI streaming
(intelligent / alive / responsive typing indicators).

## Performance perception

App must FEEL fast even during AI latency. Optimistic UI, skeleton states, streaming
responses, animated placeholders. Never freeze UI.

## Empty states

Elegant: minimal illustrations, helpful onboarding text, actionable buttons. Cases: no
history, no providers, no modes configured.

## Errors

Calm, helpful, recoverable. Never expose raw technical errors. Friendly explanation, retry
actions, diagnostics on demand.

## Keyboard-first

Every important interaction supports keyboard nav, command shortcuts, focus management,
accessibility. Most features operable without mouse.

## Accessibility

Screen readers, high contrast, reduced motion, scalable text, visible focus states.
Native, not bolted on.

## Responsive desktop

Ultrawide, small laptop, multi-monitor, DPI scaling. Windows remember size/position, open
intelligently, preserve workflow continuity.

## Loading states

Smooth indicators, shimmer, streaming feedback, progressive rendering. Never blocking
spinners unless unavoidable.

## Design consistency

Centralized design-token system. Consistent spacing, radius, iconography, predictable
layouts, coherent rhythm.

## Recommended UI stack

React, TailwindCSS, Framer Motion, shadcn/ui. Additions: Radix UI, cmdk, lucide-react.

## Windows-specific polish

Native titlebar integration, blur/transparency, startup animations, tray-native behavior,
proper DPI awareness, dark mode sync with OS, native window shadows.

## UX goal

User feels: *"This application belongs on my operating system."* Invisible when idle,
powerful when invoked, extremely responsive, thoughtfully designed.

## Final quality bar

Resemble: Raycast, Linear, Cursor, Arc Browser, modern Windows utilities.
NOT: bootstrap dashboards, generic admin panels, hobby apps. Commercially production-ready.

### Iter 1 — Theme attribute fix + backend bridge

**Problem found:** `ThemeProvider` set `<html class="light|dark">`, but the design-token
CSS (`tokens.css`) keys off `[data-theme="dark"]` / `[data-theme="light"]`. Result: light
theme didn't actually apply, and the backend `theme` / `accent` settings in the Appearance
panel didn't reach the rendered UI.

**Fix:**
1. `ThemeProvider` now writes `data-theme` (and `data-accent`) attributes on `<html>`
   instead of class names.
2. Added a `BackendThemeBridge` component that reads `useAppSettingsQuery` and pushes
   the backend `theme` and `accent` settings into the ThemeProvider whenever they
   change — so toggling theme/accent in the Appearance panel now actually re-paints
   the app.
3. Theme `system` option still respects OS dark-mode preference and listens for
   changes.

**Next likely iteration target:** Command palette polish (animation, keyboard nav,
hover states) — the spec calls it the most important UI element.
